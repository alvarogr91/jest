Angular: Tests unitarios con Jest

Tests

Un test unitario se encarga de comprobar que un fragmento de código funciona correctamente. Nos ayuda a garantizar el funcionamiento de una aplicación. Testea métodos dentro de las clases. Si necesitan hacerse dentro de otras clases, estos test se simularán.

  • Nos asegura que el código nuevo que introduzcamos no haga que deje de funcionar el resto.
  • Permite comprobar si estamos creando código demasiado complejo.

Un test de integración también incluye tests que se realizan en otras clases pero, en este caso, no serán simulados.

Un test E2E simula el comportamiento de un usuario en una aplicación.
